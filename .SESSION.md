# SESSION — Prompt Router / Model Gateway

**Last active:** 2026-05-25
**Status:** v0.2.0 — hardened with rate limiting + key isolation + BYOK

## Context
Universal LLM router — routes requests to cheapest/best model by task type. OpenAI-compatible API (drop-in replacement). Intelligent routing (8 intent classifiers), fallback chains, circuit breaker, in-memory caching, usage tracking.

## Security hardening (v0.2.0)
✅ **Key isolation** — `src/auth/key-manager.ts`: user API keys encrypted at rest (AES-256-GCM), per-request provider clients
✅ **Bring-your-own-key** — free tier (BYOK) or hosted (shared pool). `/v1/register` endpoint with tier selection
✅ **Rate limiting** — global per-IP (200 req/min) + per-user tiered limits (free: 60, pro: 200, enterprise: 1000)
✅ **Dependency audit** — 0 vulnerabilities
✅ **Auth middleware** — bearer token validation, rate limit enforcement at middleware level

## Still TODO before marketing
1. **Prompt injection testing** — need to test if malicious prompts can leak backend keys or bypass routing. Switch to OWASP skill

3. **Log sanitization** — usage logs contain prompts, need PII stripping before any production deployment
4. **HTTPS enforcement** — deploy behind Cloudflare or Railway SSL
5. **Redis for production** — rate limiting + session store currently in-memory (resets on restart)
6. **Terms of service / usage policy** — legal layer before opening to public

## Architecture decision: 
- Free tier = bring your own API key (user keys encrypted at rest, no target on your back)
- Paid tier = shared provider pool (negotiate bulk rates)
- This was the right call — free with your keys is a vulnerability magnet
